## 关键技术栈

1. **模型架构**：自研 LMNet（MobileNetV2 风格 InvertedResidual + SE + global FC head），width_mult=2.25，参数 11.54M。
2. **mean_shape 初始化**：用训练集前 512 张图估算平均脸坐标，作为 FC head 的偏置初始化，加速收敛。
3. **训练策略**：OneCycle LR（max_lr=0.002）+ AMP + grad_clip=1.0 + EMA decay=0.99。
4. **损失函数**：L1 coord loss + wing_loss（w=0.04）+ geometry loss（相邻点间距一致性）。
5. **数据增强**：随机缩放 [0.92, 1.18] + 随机平移 ±4% + 颜色抖动 + 模糊 + 随机擦除。
6. **量化**：ONNX QDQ + PTQ static + MinMax 校准，INT8 体积 11.67 MB。

## 失败实验

### `lapa_w26_hflip_e60`（w=2.6 + hflip + 60 epoch）
w=2.6（15.5M 参数）在 LaPa 18k 训练集上严重过拟合，每个 epoch 都比 w=2.25 差约 6 个百分点。60 epoch 后 test_acc_008=89.96%，比原始 w=2.25 的 92.08% 退步了 2.12pp。**结论：LaPa 18k 训练样本下，w=2.25 是最佳模型容量。**

### hflip 水平翻转
106 点翻转映射经线性分配推导并验证正确（round-trip error=0），但在当前数据量下未带来明显收益。可能原因：LaPa 18k 样本已足够丰富，翻转的边际效益有限。

## 优化瓶颈与下一步

1. **两阶段训练已验证有效**：onecycle→cosine 比单阶段多提升 0.76pp。
2. **模型容量已饱和**：w=2.25 是当前数据量下的甜点，w=2.6 过拟合。
3. **逐图片 image_acc@0.08 = 97.45%**：距 98% 仅差 0.55pp，继续提升需要更精细的方法（如 knowledge distillation、更强数据增强、或更大分辨率）。

## HRNet W18 + Heatmap 路线（2026-06-14）

### 里程碑：HRNet + Heatmap 路线首次完整训练

> 当前 HRNet 路线最佳：FP32 test NME 2.32%，acc@0.08 97.4%，image_acc@0.08 99.3%，但模型 73MB 未量化。

基于研究报告，从 LMNet 回归路线切换到 **HRNet W18 + Heatmap** 架构，全流程高分辨率特征表示 + soft-argmax 热图解码。

### LaPa 直接训练 (`lapa_hrnet_w18_heatmap_e80`)

训练配置：
- 架构：HRNet W18（base_channels=32, num_blocks=2）
- 输入：256×256，输出：106×64×64 热图
- 损失：Wing Loss (w=0.04) on heatmap
- 训练：cosine LR (0.0015) + AMP + grad_clip=1.0 + EMA (0.99) + 80 epochs
- 数据增强：hflip + scale [0.90, 1.20] + shift ±5% + crop_scale 1.30

### 精调阶段 (`lapa_hrnet_w18_heatmap_ft`)

从 best.pt (epoch 40) 加载，降低 LR (0.0015→0.0005)，增大增强幅度，EMA 0.99→0.999，加早停 (patience=15)。

**结果：29 epoch 后早停（best epoch 14），几乎无提升**

| 指标 | 原始 HRNet (best) | 精调后 (best ep14) | 变化 |
|---|---|---|---|
| test NME | 0.02324 | **0.02295** | -1.2% |

训练 loss 从 4.15→3.21 下降 23%，但 test 指标几乎不动——典型的**标注精度天花板**。
精调对 acc@0.05 有微小改善（那些 NME 在 0.05~0.08 之间的点被略微拉入阈值内），但对整体指标无实质贡献。

### INT8 量化

#### 方法一：PTQ Full（所有操作量化）❌ 退化严重

| 指标 | FP32 | INT8 Full | 退化 |
|---|---|---|---|
| NME | 0.02295 | 0.03977 | ❌ +73% |
| acc@0.08 | 97.35% | 92.74% | ❌ -4.61pp |
| acc@0.05 | 92.15% | 74.61% | ❌ -17.54pp |
| image_acc@0.08 | 99.15% | 98.35% | ⚠️ -0.80pp |
| 模型大小 | 69.7 MB | 17.82 MB | ✓ |

退化原因：heatmap→softmax→soft-argmax 量化链非线性太强，softmax 的指数放大效应使 INT8 精度的误差被放大为坐标偏移。

#### 方法二：PTQ Conv-Only（只量化 Conv，保留解码链 FP32）✅ 可用

| 指标 | FP32 | INT8 Conv-Only | 退化 |
|---|---|---|---|
| NME | 0.02295 | **0.02560** | ✅ +11.5% |
| acc@0.05 | 92.15% | **90.70%** | ✅ -1.45pp |
| acc@0.08 | 97.35% | **97.03%** | ✅ -0.32pp |
| acc@0.10 | 98.48% | **98.31%** | ✅ -0.17pp |
| image_acc@0.08 | 99.15% | **99.00%** | ✅ -0.15pp |
| 模型大小 | 69.7 MB | **17.93 MB** | ✓ 5~20MB |

**策略**：只量化 53 个 Conv 操作（backbone 权重），保留 Softmax + soft-argmax + 算术操作为 FP32。Percentile 校准 (99.999%)，32 calibration batches。

**结论**：conv_only 量化大幅修复了退化问题，acc@0.08 退化从 -4.61pp 收敛到 -0.32pp，image_acc@0.08 = 99.0% 仍然超过 98% 目标。模型 17.93MB 在 5~20MB 区间内。

**关键对比（HRNet vs LMNet）**：

| 指标 | LMNet w2.25 (FP32) | HRNet W18 (FP32) | 提升 |
|---|---|---|---|
| test NME | 0.0376 | **0.0232** | **-38.3% 相对下降** |
| test acc@0.08 | 92.86% | **97.35%** | **+4.49pp** |
| test image_acc@0.08 | 97.50% | **99.25%** | **+1.75pp** |

**核心判断**：
- HRNet + Heatmap 路线**大幅领先** LMNet 回归路线，NME 相对下降 38%。
- image_acc@0.08 = 99.3%，**已超过 98% 目标**。
- acc@0.08 = 97.4%，接近 98% 目标，仅差 0.6pp。
- 训练在 epoch 43 中断，后 37 个 epoch 未跑完，后期 plateau，继续训练收益有限。
- FP32 模型 73MB 偏大，需量化压缩至 5~20MB 区间。

### WFLW 预训练 (`wflw_hrnet_w18_pretrain_e40`)

- 配置：68 点 WFLW → 40 epochs
- 进度：29 / 40 epochs（中断）
- 最佳 test NME = 0.1355，acc@0.08 = 61.6%
- **结论**：WFLW 68 点任务远未收敛，训练效果不理想。WFLW→LaPa 的迁移预训练路径暂时搁置。

### 混合数据训练 + AWing Loss (`lapa_hrnet_w18_awing_mixed_e80`)

从 HRNet best checkpoint (ep43) 加载，使用 Adaptive Wing Loss + 113k 混合数据：
- LaPa 18k 人工标注 + JD-landmark 20k 人工标注 + PseudoWFLW 75k 伪标签（HRNet teacher）
- cosine LR 0.0015 → 1e-6，EMA 0.999，早停 patience=20

**结果：80 epoch 全量训练，best epoch 78**

| 指标 | 旧 HRNet best (ep14 ft) | 混合训练 (ep78) | 改善 |
|---|---|---|---|
| test NME | 0.02295 | **0.02164** | **-5.7%** |
| test acc@0.08 | 97.35% | **97.98%** | **+0.63pp** |
| test acc@0.05 | 92.15% | **93.11%** | +0.96pp |
| image_acc@0.08 | 99.15% | **99.55%** | +0.40pp |

**训练曲线特征**：
- ep1~10: NME 0.0390 → 0.0264（快速下降）
- ep10~41: NME 0.0264 → 0.0230（稳定改善）
- ep41~55: NME 0.0230 → 0.0222（plateau + 间歇突破）
- ep55~78: NME 0.0222 → 0.0216（cosine LR 后期精细优化）
