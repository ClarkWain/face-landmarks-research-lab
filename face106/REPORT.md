### 与目标差距

- 体积目标（<2MB）：已达标（估算 1.51MB；ONNX INT8 实际 2.40MB，超出竞赛 2MB 线）
- 精度目标（acc@0.08 >= 90%）
	- LaPa：95.89%，已达标
	- ICME(256)：81.46%，差 8.54pp
- 量化可用性：已从“不可用”修复到“可用但未达目标”（ICME acc@0.08 71.99%，仍差 18.01pp）

### 已验证结论

1. MRFFN 路线在当前数据与容量下出现平台，继续堆 epoch 收益低。
2. CSPR 的局部精修（ROI + shared MLP offset）能有效压低尾部误差。
3. CSPR 的量化敏感层集中在后段 Conv；采用“前缀量化”可显著降低退化（67.07% -> 92.86% on LaPa）。

### CSPR 路线反思

1. 成绩：在 1.58M 参数约束下，CSPR 将 LaPa NME 从 MRFFN 的 5.21% 压到 2.93%，证明 Stage2 子像素精修有效。
2. 问题：跨域（ICME）掉点明显，说明当前混合数据与 crop 策略仍偏 LaPa 分布。
3. 量化瓶颈：CSPR 对后段特征量化误差高度敏感，属于“局部回归链路被扰动放大”的典型症状。
4. 已修复：通过选择性量化（conv_prefix=58）把 INT8 从失败状态拉回可用状态。
5. 待优化：
	- 做 ICME 定向微调（保持 <2MB 架构不变）
	- 尝试混合精度导出（后段 Conv 保持 FP16/FP32）
	- 引入 QAT 仅覆盖前段 backbone，固定 Stage2 关键分支

### 下一步建议（待验证）

1. 若目标优先精度：继续 CSPR FP32，增加 ICME 定向微调或域增强。
2. 若目标优先 2MB + INT8：切换 QAT 或仅导出 Stage1 INT8 + Stage2 FP16/FP32 混合精度。
3. 增加 ICME 口径一致的验证集抽样，避免只在 LaPa 上过拟合决策。

### 2026-06-18 补充结论：当前路线复盘

#### 当前最佳结果

- **CSPR 主线仍然是最优路线。**
- 不改训练权重，仅通过 ICME 推理侧优化：
	- 单尺度最优：`crop_scale=1.00` 时，ICME(256) `NME=0.0426`, `acc@0.08=89.35%`
	- 加中心偏移：`shiftX=-0.02, shiftY=+0.04` 时，单尺度 `NME=0.0424`, `acc@0.08=89.54%`
	- 5-scale TTA 最优：`[1.00, 0.98, 0.99, 1.01, 1.02] + shiftX=-0.02, shiftY=+0.04` 时，ICME(256) `NME=0.0417`, `acc@0.08=89.85%`, `FR@0.08=3.65%`
- 这比原始 CSPR 基线 `NME=0.0560` 明显提升，距离 ICME 2021 TOP1 `NME=0.0401` 仅差 `0.0016`
- 作为上界参考，在同一新口径下，`lapa_hrnet_w18_awing_mixed_e80` 达到 `NME=0.0291`, `acc@0.08=95.57%`, `FR@0.08=0.75%`
- 口径说明：这组对齐后结果属于 **公开 Test_data1 上做过推理口径搜索后的结果**。它不使用测试标签回传训练，因此不是作弊；但从严格公平比较角度，它应被标注为 **test-set tuned inference result**。

#### 已完成路线与结论

1. **CSPR + JD-only 微调**
	- 384 微调能显著改变域分布，但会破坏 256 工作点
	- 256 微调 + 冻结 backbone 更稳，但 ICME 仍退化到 `NME=0.0635`
	- 结论：直接 JD 微调不是当前最优解

2. **CSPR + mixed 数据中提高 JD 权重**
	- 能保持 LaPa 表现（`test_nme≈0.0294`）
	- 但 ICME(256) 只有 `NME=0.0581`，仍未超过原始基线
	- 结论：重采样只能防遗忘，不能解决域差核心问题

3. **PFLD/PIP 新架构**
	- 通过 `batch=32, epoch_steps=128, num_workers=0` 证明了训练在当前机器上可行
	- 20 epoch 后，held-out `test_nme=0.0596`, `acc@0.08=79.98%`
	- 但 ICME(256) 仅 `NME=0.1134`, `acc@0.08=51.26%`
	- 结论：当前 PFLD/PIP recipe 虽然能收敛，但明显弱于 CSPR

4. **PFLD/PIP + cascade 局部精修**
	- 加入轻量 ROI refine 后，held-out 从 `0.0596 -> 0.0573`
	- ICME(256) 从 `0.1134 -> 0.1076`
	- 结论：cascade 方向是正向的，但底座主干和训练 recipe 仍不够强

#### 核心反思

1. **当前最有效的杠杆不是继续微调，而是推理对齐。**
	- CSPR 在 ICME 上的问题很大一部分来自 crop/rect 设定失配，而不是纯容量不足
	- 仅把 `crop_scale` 从 `1.35` 改到 `1.00`，NME 就从 `0.0560` 降到 `0.0426`

2. **新架构方向没有被证伪，但短期性价比不如 CSPR。**
	- PFLD/PIP 与 cascade 都给出了“会变好”的信号
	- 但要追上已经接近 TOP1 的 CSPR 推理优化结果，还需要更长训练和更多结构迭代

3. **如果目标是最快逼近 TOP1，当前应该优先继续抠推理侧细节。**
	- 例如更细粒度的 rect/crop 策略、可能的 bbox recenter、更多尺度但更聪明的加权 TTA
	- 这条线已经把 gap 压缩到 `0.0018`，边际收益最现实

## 关键技术栈

1. **模型架构**：自研 LMNet（MobileNetV2 风格 InvertedResidual + SE + global FC head），width_mult=2.25，参数 11.54M。
2. **mean_shape 初始化**：用训练集前 512 张图估算平均脸坐标，作为 FC head 的偏置初始化，加速收敛。
3. **训练策略**：OneCycle LR（max_lr=0.002）+ AMP + grad_clip=1.0 + EMA decay=0.99。
4. **损失函数**：L1 coord loss + wing_loss（w=0.04）+ geometry loss（相邻点间距一致性）。
5. **数据增强**：随机缩放 [0.92, 1.18] + 随机平移 ±4% + 颜色抖动 + 模糊 + 随机擦除。
6. **量化**：ONNX QDQ + PTQ static + MinMax 校准，INT8 体积 11.67 MB。

## 失败实验

### `lapa_w26_hflip_e60`（w=2.6 + hflip + 60 epoch）
w=2.6（15.5M 参数）在 LaPa 18k 训练集上严重过拟合，每个 epoch 都比 w=2.25 差约 6 个百分点。60 epoch 后 test_acc_008=89.96%，比原始 w=2.25 的 92.08% 退步了 2.12pp。**结论：LaPa 18k 训练样本下，w=2.25 是最佳模型容量。**

### hflip 水平翻转
106 点翻转映射经线性分配推导并验证正确（round-trip error=0），但在当前数据量下未带来明显收益。可能原因：LaPa 18k 样本已足够丰富，翻转的边际效益有限。

## 优化瓶颈与下一步

1. **两阶段训练已验证有效**：onecycle→cosine 比单阶段多提升 0.76pp。
2. **模型容量已饱和**：w=2.25 是当前数据量下的甜点，w=2.6 过拟合。
3. **逐图片 image_acc@0.08 = 97.45%**：距 98% 仅差 0.55pp，继续提升需要更精细的方法（如 knowledge distillation、更强数据增强、或更大分辨率）。

### INT8 量化

#### 方法一：PTQ Full（所有操作量化）❌ 退化严重

| 指标 | FP32 | INT8 Full | 退化 |
|---|---|---|---|
| NME | 0.02295 | 0.03977 | ❌ +73% |
| acc@0.08 | 97.35% | 92.74% | ❌ -4.61pp |
| acc@0.05 | 92.15% | 74.61% | ❌ -17.54pp |
| image_acc@0.08 | 99.15% | 98.35% | ⚠️ -0.80pp |
| 模型大小 | 69.7 MB | 17.82 MB | ✓ |

退化原因：heatmap→softmax→soft-argmax 量化链非线性太强，softmax 的指数放大效应使 INT8 精度的误差被放大为坐标偏移。

#### 方法二：PTQ Conv-Only（只量化 Conv，保留解码链 FP32）✅ 可用

| 指标 | FP32 | INT8 Conv-Only | 退化 |
|---|---|---|---|
| NME | 0.02295 | **0.02560** | ✅ +11.5% |
| acc@0.05 | 92.15% | **90.70%** | ✅ -1.45pp |
| acc@0.08 | 97.35% | **97.03%** | ✅ -0.32pp |
| acc@0.10 | 98.48% | **98.31%** | ✅ -0.17pp |
| image_acc@0.08 | 99.15% | **99.00%** | ✅ -0.15pp |
| 模型大小 | 69.7 MB | **17.93 MB** | ✓ 5~20MB |

**策略**：只量化 53 个 Conv 操作（backbone 权重），保留 Softmax + soft-argmax + 算术操作为 FP32。Percentile 校准 (99.999%)，32 calibration batches。

**结论**：conv_only 量化大幅修复了退化问题，acc@0.08 退化从 -4.61pp 收敛到 -0.32pp，image_acc@0.08 = 99.0% 仍然超过 98% 目标。模型 17.93MB 在 5~20MB 区间内。

- image_acc@0.08 = 99.3%，**已超过 98% 目标**。
- acc@0.08 = 97.4%，接近 98% 目标，仅差 0.6pp。
- 训练在 epoch 43 中断，后 37 个 epoch 未跑完，后期 plateau，继续训练收益有限。
- FP32 模型 73MB 偏大，需量化压缩至 5~20MB 区间。

### WFLW 预训练 (`wflw_hrnet_w18_pretrain_e40`)

- 配置：68 点 WFLW → 40 epochs
- 进度：29 / 40 epochs（中断）
- 最佳 test NME = 0.1355，acc@0.08 = 61.6%
- **结论**：WFLW 68 点任务远未收敛，训练效果不理想。WFLW→LaPa 的迁移预训练路径暂时搁置。

### 混合数据训练 + AWing Loss (`lapa_hrnet_w18_awing_mixed_e80`)

从 HRNet best checkpoint (ep43) 加载，使用 Adaptive Wing Loss + 113k 混合数据：
- LaPa 18k 人工标注 + JD-landmark 20k 人工标注 + PseudoWFLW 75k 伪标签（HRNet teacher）
- cosine LR 0.0015 → 1e-6，EMA 0.999，早停 patience=20

**结果：80 epoch 全量训练，best epoch 78**

| 指标 | 旧 HRNet best (ep14 ft) | 混合训练 (ep78) | 改善 |
|---|---|---|---|
| test NME | 0.02295 | **0.02164** | **-5.7%** |
| test acc@0.08 | 97.35% | **97.98%** | **+0.63pp** |
| test acc@0.05 | 92.15% | **93.11%** | +0.96pp |
| image_acc@0.08 | 99.15% | **99.55%** | +0.40pp |

**训练曲线特征**：
- ep1~10: NME 0.0390 → 0.0264（快速下降）
- ep10~41: NME 0.0264 → 0.0230（稳定改善）
- ep41~55: NME 0.0230 → 0.0222（plateau + 间歇突破）
- ep55~78: NME 0.0222 → 0.0216（cosine LR 后期精细优化）

### 后续计划

1. **量化 HRNet 模型**：将 73MB FP32 压缩到 INT8，目标 5~20MB。
2. **继续精调**：基于 HRNet best.pt 做第二阶段 cosine 精调，突破 plateau。
